# ADR-003 — Бренд-архитектура: один сайт, два бренда

## Контекст

Фабрика «Исток-мебель» производит три типа товаров под двумя торговыми марками:

- **«Исток-мебель»** (головной бренд) — театральные кресла (B2B/госзаказ, ~1/3 рынка
  залов РБ по заявлению клиента) и корпусная мебель (комоды, столы, стеллажи, шкафы).
- **«ELIS-MEBEL Kids Beds»** (суббренд) — 8 моделей детских кроваток (Николь, Мелита,
  Паулина, Эльза, Доминика, Афина, Ясмина, Мишель), аудитория — родители (B2C).

В PDF-каталоге суббренд оформлен как самостоятельная марка: собственная бежево-кремовая
палитра, мягкая типографика, эмоциональный тон для родителей. Головной бренд — деловой,
B2B-ориентированный. Аудитории и задачи принципиально разные.

На текущем Tilda-сайте `istokmebel.by` детские кроватки существуют как раздел — отдельного
домена или сайта для ELIS никогда не было. Решение «делать ли отдельный сайт или нет» ранее
не формализовалось; теперь зафиксируем явно, чтобы архитектура и дизайн-система строились
с правильными предпосылками.

## Варианты

### A. Один сайт, единая навигация — без выраженного суббренда

- Плюсы: простейшая IA, один SEO-домен, один деплой, минимум абстракций в коде.
- Минусы: ELIS теряется как бренд. Маркетинговое послание для родителей растворяется
  в общем B2B-тоне «Исток». Упускается возможность строить B2C-воронку для суббренда
  через узнаваемый визуальный язык.

### B. Два отдельных сайта / домена (istokmebel.by + elis-mebel.by или аналог)

- Плюсы: чистое позиционирование для каждой аудитории, независимые SEO- и SMM-стратегии.
- Минусы: два деплоя, два PocketBase (или сложная мультиарендность), дублирование
  контактов и юридических данных, разделение SEO-сигнала между доменами, двойная
  операционная нагрузка. Пользователь явно отверг этот вариант в ходе обсуждения:
  «у Элиса нет сайта, это чисто название».

### C. Один сайт `istokmebel.by`, ELIS — раздел с собственной визуальной темой

- Плюсы: один деплой и одна инфраструктура, юридически единое целое. При этом раздел
  `/elis` имеет собственную палитру, типографику и эмоциональный тон через систему
  CSS-токенов — ощущение «отдельного мира для родителей» без удвоения инфры. Один
  SEO-домен с чёткой структурой `/elis/<model>` накапливает авторитет единого домена.
  Архитектура токенов позволяет добавлять дополнительные тематические разделы (например,
  `/goszakaz`) без рефакторинга.
- Минусы: требует дисциплины в дизайн-системе — две темы внутри одной кодобазы.
  Небольшой риск размытия бренда «Исток» при чрезмерной автономии раздела; нейтрализуется
  явными дизайн-гайдами.

## Решение

Выбираем **C**: один сайт `istokmebel.by`, ELIS-MEBEL Kids Beds — как раздел с
собственной визуальной темой.

Юридически это одна компания с едиными контактами и реквизитами — делить инфраструктуру
нет экономического смысла. Один SEO-домен с хорошо структурированными URL
(`/elis`, `/elis/<slug-модели>`) и правильным JSON-LD накапливает авторитет быстрее, чем
два слабых домена. Механизм переключения темы по layout-у роута (`theme.brand-elis` через
CSS custom properties + cascade layer) даёт маркетинговый эффект «своего пространства»
без удвоения деплоя, поддержки и контент-команды.

Если в будущем маркетинговая стратегия потребует отдельного домена для ELIS — это
реализуемо через ребрендинг раздела и 301-редиректы со старых URL. Данные уже
структурированы под это (relation `products → brand`). Решение обратимо — переезд
несёт SEO-просадку, но не катастрофическую.

## Последствия

**Что делаем:**

- Один Next.js-проект, один деплой, один PocketBase.
- В дизайн-системе закладываем **механизм тематических подразделов**: токены палитры
  и типографики переопределяются на уровне layout-а раздела `/elis` через CSS custom
  properties + cascade layer `theme.brand-elis`. Закладываем сразу — не дописываем потом.
- Навигация: на главной `istokmebel.by` блок «Детские кроватки ELIS» ведёт в раздел.
  В шапке раздела ELIS — перекрёстная ссылка обратно на головной сайт.
- SEO: единый `sitemap.xml`, общий JSON-LD `Organization`. В разделе `/elis/<model>` —
  отдельные `Product` + `Brand` JSON-LD с `brand: "ELIS-MEBEL Kids Beds"`.
- Бренд-документация: `docs/65 - Brand/` организована в две поддиректории — `istok/` и
  `elis/` — каждая со своими identity, palette, typography, voice-of-brand.
- Схема PocketBase: коллекция `brands` (записи «Исток», «ELIS»), коллекция `products`
  ссылается на `brand` через relation. Категория товара — отдельное поле/коллекция
  (`chairs`, `cabinets`, `cribs`).
- Финальный slug раздела (`/elis`, `/krovatki` или эквивалент) — решается в ADR по
  информационной архитектуре (будущий ADR).

**Что НЕ делаем:**

- Не покупаем домен `elis-mebel.by` / `eliskidsbeds.by` и аналоги в рамках текущего
  этапа. Вариант остаётся доступной опцией — при смене маркетинговой стратегии
  оформляется новым ADR.
- Не создаём второй PocketBase и второй фронтенд для ELIS.
- Не разводим лиды из ELIS и «Исток» по разным CRM/Telegram-чатам в Этапе 1.
  Допустимо тегирование `source: istok|elis` внутри одной системы — если потребуется,
  оформляется отдельным ADR.

**Обратимость: лёгкая.**

Выделить ELIS на отдельный домен в будущем — переезд через ребрендинг раздела +
301-редиректы со старых URL. SEO-просадка ожидаема, но контролируема; PocketBase
обеспечивает чистый экспорт данных. Архитектура токенов позволяет добавлять новые
тематические разделы (например, отдельная тема для госзаказа) без рефакторинга
дизайн-системы.

**Кто затронут:**

- **Разработка** — закладывает механизм тематических подразделов в дизайн-систему
  с первого дня, а не дописывает его при появлении ELIS-страниц.
- **Бренд / маркетинг** — ведёт два бренд-гайда (`docs/65 - Brand/istok/` и
  `docs/65 - Brand/elis/`) с самого начала.
- **SEO** — два набора JSON-LD `Brand` (Исток / ELIS), один `Organization`, один sitemap.
- **Контент-команда** — при заведении товаров в PocketBase выбирает бренд через
  relation-поле (dropdown с двумя значениями).

## Имплементация

Не начата. Зависит от ADR-002 (Next.js 15, App Router layouts) и ADR-004 (SCSS modules +
token-система). После их принятия:

1. Создать `docs/65 - Brand/istok/` и `docs/65 - Brand/elis/` — перенести/задокументировать
   палитру и типографику обоих брендов.
2. В PocketBase добавить коллекцию `brands` и relation в `products`.
3. В Next.js реализовать layout для роута `/elis` (или выбранного slug-а) с применением
   `theme.brand-elis` CSS custom properties.
