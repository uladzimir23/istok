# Каталог — спецификация полей и план данных

## 1. Зачем этот документ

Перед тем как переносить каталог фабрики на новый сайт, нужно зафиксировать:

1. **Какую структуру** имеет карточка товара (поля + типы данных).
2. **Что у нас уже есть** в текущем placeholder-каталоге (после миграции
   с Tilda и парсинга PDF Элис).
3. **Что критично запросить у клиента** — структурированный чек-лист
   с приоритетами.

Документ согласуется с фабрикой → закрепляется как источник истины →
после этого мы заполняем MDX-файлы и публикуем разделы.

> **Подход MVP-first.** Базовые тексты, разметку и черновики характеристик
> мы заполняем сами по фото и существующим каталогам. Финальные значения
> (точные размеры, материалы, цены) — от клиента.

## 3. Структура карточки товара (поля)

Все поля описаны в Zod-схеме `src/entities/product/schema.ts`. Ниже —
человеческое описание для согласования.

| Группа | Поле | Тип / формат | Обязательное | Кто заполняет | Пример |
| --- | --- | --- | --- | --- | --- |
| **Идентификация** | `slug` | строка `a-z0-9-` | да | мы | `m3-1` |
|  | `name` | строка | да | мы (черновик), клиент проверяет | `Кресло М3-1 «Премиум»` |
|  | `category` | `chairs` `\|` `cabinets` `\|` `cribs` | да | мы | `chairs` |
|  | `brand` | `istok` `\|` `elis` | да | мы | `istok` |
|  | `summary` | до 220 симв. | да | мы (черновик) → клиент правит | «Топовая модель театрального кресла. Высокая спинка, мягкое сиденье, …» |
| **Размеры** | `sizes[]` — массив вариантов | объект с `slug` + `totalDimensions` | да | **клиент** | для кроваток — несколько размеров; для кресел — обычно один |
|  | `totalDimensions` | мм: длина / ширина / высота | да | **клиент** | `1680 × 875 × 720 мм` |
|  | `bedDimensions` | для кроваток — спальное место мм | если cribs | **клиент** | `1600 × 800 мм` |
|  | `weight` | кг | желательно | **клиент** | `23` |
| **Материалы** | `materials[]` | массив строк | да | **клиент** | «Берёзовая фанера», «MDF 16мм», «Поролон 35кг/м³», «Ткань Velure», «Подлокотники: бук массив» |
| **Палитра** | `colors[]` | массив `{ name, hex }` | да | **клиент** | `{ name: "Бордо", hex: "#7C1F2C" }` |
| **Опции** | `options[]` | массив строк | да | **клиент** | «Откидное сиденье», «Подлокотники», «Нумерация места», «Лого на спинке» |
| **Медиа** | `hero` | один MediaItem | да | мы (из Tilda/PDF) → клиент даёт hi-res | главное фото товара |
|  | `gallery[]` | массив MediaItem | да (мин 1) | мы | дополнительные фото |
|  | `MediaItem.type` | `photo` `\|` `render` `\|` `diagram` | — | — | сейчас все = `photo`, чертежи нужны |
| **Коммерция** | `priceByn` | целое BYN | да | **клиент** | `380` |
|  | `priceNote` | строка (опц.) | нет | мы | «от» / «за единицу» / «без обивки» |
| **Публикация** | `published` | bool | да | мы | `true` если показывать |
|  | `indexable` | bool | да | мы | `false` для draft'ов / WIP |
|  | `updatedAt` | YYYY-MM-DD | да | мы | при каждом изменении |

### Что **отсутствует** в текущей схеме и нужно добавить

| Поле | Зачем | Кто решает |
| --- | --- | --- |
| `subCategory` для cabinets | разделить komody/stoly/stelazhi/shkafy явно (сейчас выводится через хелпер) | мы |
| `moq` — минимальный заказ | важно для B2B | клиент |
| `leadTime` — срок производства | «28–45 дней», варьируется по модели | клиент |
| `certificates[]` — привязка сертификатов | для тендерных карточек | клиент |
| `tags[]` — теги для поиска и фильтров | «огнестойкая», «антивандальная», «детская», «театральная» | мы |
| `diagram` в gallery | технический чертёж с размерами | клиент или мы (рисуем) |

> Эти поля добавим в schema **после** согласования.

## 5. Что просим у клиента — чек-лист

### 5.1 Кресла · приоритет HIGH

Для каждой из 6 моделей (М1, М2, М3, М3-1, М3-Г, ПМ-1):

- [ ] **Полное название** (как у вас в прайсе)
- [ ] **Размеры** — длина × ширина × высота в мм для одного посадочного места + (если есть) межосевое расстояние / ширина с подлокотниками
- [ ] **Вес** одного кресла, кг
- [ ] **Материалы** — каркас (металл/дерево/комбинация), обивка (тип ткани, плотность поролона), фурнитура (механизм откидывания и т.п.)
- [ ] **Палитра** — список доступных цветов обивки (название + образец или hex по RAL/Pantone — 4–8 вариантов на модель достаточно)
- [ ] **Опции комплектации** — откидное сиденье да/нет, подлокотники да/нет, нумерация места/ряда (металл/пластик/нет), логотип на спинке, варианты задней панели
- [ ] **Цена за единицу в BYN** на текущий момент (с пометкой «от»/«за серию X+ единиц»)
- [ ] **МОЗ** — минимальный заказ для серийной поставки
- [ ] **Срок производства** — диапазон по модели
- [ ] **Технический чертёж** (PDF/JPG) — если есть. Если нет — отметим, нарисуем сами по фото и согласуем

**Дополнительно:**

- [ ] Есть ли модели вне этого списка, которые сейчас в производстве?
  (если убраны / добавлены — обновим)
- [ ] Высококачественные фото каждой модели (если нынешние с Tilda
  низкого разрешения) — минимум 1 «3/4», 1 «прямо», 1 «деталь»

### 5.2 Корпусная мебель · приоритет HIGH

Сейчас в каталоге 12 моделей, **все — шкафы и комоды**. Столов и стеллажей нет.

- [ ] **Уточнить актуальный модельный ряд** по подкатегориям:
  - Шкафы — сколько моделей, какие
  - Комоды — сколько моделей, какие
  - Столы — есть ли вообще в текущем производстве?
  - Стеллажи — есть ли?
- [ ] Если разделов «Столы» / «Стеллажи» нет — подтвердить,
      что временно убираем из навигации
- [ ] Для каждой модели (по тому же шаблону что и кресла):
  название · размеры · вес · материалы · палитра · опции · цена ·
  МОЗ · срок · чертёж · фото

> Возможный быстрый формат для корпусной — **одна Excel-таблица** со
> столбцами «slug · название · размеры · цена …» — все 12 модели
> на одном листе.

### 5.3 Кроватки ELIS · приоритет MED

Для каждой из 8 моделей:

- [ ] **Палитра** — какие цвета берёзы / лака / эмали доступны.
      Образцы фото или RAL-номера
- [ ] **Цена за каждый размер спального места** — BYN
- [ ] **МОЗ** — кроватки розница, минимум 1 шт, или есть скидка от партии?
- [ ] **Срок производства** для розничного заказа
- [ ] **Соответствие имени и кодов ELIS** — например, «Афина» = «Элис-6».
      Эта связка важна для SEO и для команды (артикул в 1С). Прислать полную таблицу

### 5.4 Общее по фабрике · приоритет HIGH

- [ ] **Сертификаты** — копии или сканы (СТБ, ISO, пожарная безопасность,
      гигиена). У нас на сайте сейчас placeholder-блок «Тендерный комплект»
      с 4 ячейками — нужны реальные номера и сроки действия
- [ ] **Реальные проекты-портфолио** — у нас сейчас 6 placeholder-карточек
      (Минск/Гомель/Брест/Витебск/Могилёв/Гродно). Нужны:
  - название объекта (анонимно или открыто — по согласованию)
  - город · год поставки
  - 1–3 фото из зала
  - количество поставленных кресел/изделий
  - модель кресла, цвет
- [ ] **Тексты «О фабрике»** — основа сейчас собрана нами, нужно
      проверить и дополнить:
  - точная дата основания (у нас «2008 г.»)
  - количество сотрудников
  - площадь производства (м²)
  - количество производственных линий
  - 1–2 цитаты для блока «Команда / Слово директора» (опционально)

## 6. Формат предоставления данных

Вариант на выбор клиента:

| Формат | Когда удобно | Кто делает |
| --- | --- | --- |
| **A) Excel/Google Sheets** по нашему шаблону | если есть менеджер по каталогу — самый быстрый путь | мы готовим шаблон, клиент заполняет |
| **B) Заполнение через нас (созвон + текстовка)** | если данных мало или они «в голове» | клиент рассказывает, мы заполняем |
| **C) Передача исходников** (старые прайсы, PDF, фото-архив) | если структурированных данных нет | мы парсим, согласуем |

Рекомендуем **B + C**: одна встреча на категорию (45–60 мин), мы заполняем
по ходу, клиент даёт расходные материалы (старые прайсы, фото из 1С, образцы
тканей).

## 7. Подкатегории корпусной — структура

Сейчас в навигации заявлены 4 подкатегории. Текущее распределение:

| Подкатегория | URL | Моделей |
| --- | --- | --- |
| Шкафы | `/shkafy/` | 4 |
| Комоды | `/komody/` | 8 |
| Столы | `/stoly/` | 0 (?) |
| Стеллажи | `/stelazhi/` | 0 (?) |

**Решение к согласованию:**

- Если столы/стеллажи **есть в производстве** — клиент даёт данные,
  поднимаем разделы
- Если **нет** — временно скрываем из меню / переключателя
  (`published: false` на уровне подкатегории)
