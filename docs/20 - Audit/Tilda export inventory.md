# Инвентаризация Tilda export

> **TL;DR.** Получили полный экспорт сайта с Tilda: 50 HTML-страниц, 151 МБ
> изображений, 11 МБ файлов. Это **главный источник реального контента** для
> Phase 1: тексты товаров, фото, описания, метаданные, URL'ы. Папка
> `tilda-old/` в gitignore (большая статика); сама структура и URL-маппинг
> зафиксированы в этом файле и в `docs/40 - Architecture/Карта URL.md`.

## Стек Tilda (детектировано из HTML)

- **Шрифты:** Montserrat (UI) + Comfortaa (акцент). Cyrillic + Latin subsets.
- **Yandex Verification:** `708ff1a7f9fdbfe1` (нужно сохранить при cutover).
- **CDN ассетов:** `tildacdn.com`.
- **Аналитика на сайте:** не обнаружена (ни Метрики, ни GA4, ни пикселей в HTML).

## Как Tilda хранит контент в HTML

- `page-NNN.html` — полная страница с `<head>`, навигацией, телом, футером.
- `files/page-NNNbody.html` — тело страницы без header/footer (~30% меньше),
  удобнее для парсинга.
- Метаданные товара (title, og:title, og:description, canonical) — в `<head>`
  каждой страницы.
- Цены — нет (на проверенных страницах). Похоже, цены не отображались публично.
- Описания товара — текстовые блоки внутри Tilda-классов (`.t1234__title`,
  `.t1234__descr` и пр.). Парсятся регулярно.

Большинство страниц моделей не имеют `meta description` — это явная SEO-проблема,
которую в новом сайте закрываем (генерация description из первого абзаца + cap 155).

## Что делаем с этим экспортом

**Phase 1 имплементация:**

1. **Парсер `scripts/parse-tilda.ts`** (отдельная задача) — пройдётся по всем
   `page-*.html`, извлечёт через cheerio: `title`, описания, фото-список,
   галерею. Сгенерирует `content/products/*.mdx` по Zod-схеме.
2. **Изображения** — копировать в `web/public/images/<category>/<sku>/`,
   переименовывать по схеме `01.jpg, 02.jpg, ...`.
3. **URL-маппинг** — в `nginx/sites-available/istokmebel.conf` (см. [[Карта URL]]).

**Что НЕ переносим:**

- CSS/JS Tilda — не нужны (новая дизайн-система с нуля).
- Yandex Verification — переподтвердим в Search Console (можно сохранить старый
  токен на новом сайте для быстрой ре-верификации).
- Служебные страницы (`/headermobile`, `/footer`, `/test`) — 404.
