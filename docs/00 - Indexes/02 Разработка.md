## ADR landscape

(пусто на момент init)

Ожидаемые в Phase 1:
- ADR-001 — PocketBase as backend (rationale: ≤500 SKU, готовая админка, single binary)
- ADR-002 — Next.js 15 + App Router (rationale: SEO, SSR, ISR-ревалидация, ниша)
- ADR-003 — «Элис» как раздел внутри istokmebel.by, не отдельный сайт/домен
- ADR-004 — Hosting (self-host Docker на VPS vs Vercel — выбрать)
- ADR-005 — UI стек (Tailwind+shadcn vs SCSS modules — выбрать)
- ADR-006 — Image pipeline (R2 vs Bunny + next/image)
- ADR-007 — Аналитика (Метрика + GA4 + цели)
- ADR-008 — AI-рендер пайплайн (Gemini/Krea пресеты, batch-обработка)

## Pre-commit / CI

(не настроены на момент init)

Планируем:
- `scripts/check-secrets.sh` — regex-сканер секретов (см. `.claude/skills/redaction.md`).
- `lefthook` или `husky` — pre-commit hook.
- GH Actions: typecheck + lint + build на PR; deploy на push в `main` (после Phase 4).

## Риски

🟡 **Контент-зависимость от клиента.** Без цен и каталога кресел сайт не запустится
полноценно. Параллельно с разработкой — бить на доставке контента.

🟡 **AI-пайплайн не формализован.** У клиента есть рендеры из Gemini/Krea, но без
пресетов и системы. Нужен ADR + playbook.

🟡 **Tilda как «работающий блокер».** Cutover требует подготовки DNS, тестов, окна
простоя — отдельная фаза с чек-листом.
