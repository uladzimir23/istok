# Архитектура знаний проекта istok

> **TL;DR.** В проекте 7 слоёв информации — от памяти AI-агента до immutable снапшотов.
> Они дополняют друг друга, а не дублируют. Эта страница — карта, чтобы за 5 минут
> понять «что где искать» и «куда писать новое».

## Карта слоёв

| Слой | Где лежит | Кто читает | Что хранит | Immutable? |
|---|---|---|---|---|
| **CLAUDE.md** | Корень репо | Claude + человек | Brief проекта, стек, redaction policy, видение | Нет (живёт) |
| **Dashboard** | `docs/00 - Indexes/Dashboard.md` | Все роли | Текущий статус, блокеры, ссылки на ключевое | Нет (живёт) |
| **Dept-dashboards** | `docs/00 - Indexes/01 Маркетинг.md`, `02 Разработка.md` | Отделы | Статус на языке конкретной аудитории | Нет (живёт) |
| **Документация** | `docs/10..97/**/*.md` | Люди + Claude по запросу | ADR, session logs, briefs, audits, planы | Частично (session logs + ADR редко правятся) |
| **Снапшоты** | `docs/70 - Meetings & Logs/snapshots/<slug>/` | Будущие модели, отчёты, debug | Frozen копия индексов+памяти+git-log на момент T | Да |
| **Скилы** | `.claude/skills/*.md` | Claude | Процедурные правила | Нет (живёт) |
| **Агенты** | `.claude/agents/*.md` | Claude (spawn) | Workers со scoped-правами (docs-sync, adr-drafter) | Нет (живёт) |
| **Память** | `~/.claude/projects/-Users-vladimirmazyrec-Documents-istok/memory/*.md` | Claude (auto-load) | Факты last-known-state для будущих сессий | Нет (перезаписывается) |

### Уровни изменяемости

```
immutable (git-history + snapshots)
   ↑
   │ ADR (правятся редко, обычно только статус)
   │ session logs (не правятся после записи)
   │ Dashboard, dept-dashboards (правятся постоянно)
   │ CLAUDE.md (правится при сдвигах фокуса)
   │ память (правится каждую сессию)
   ↓
mutable (current state)
```

Снапшоты — механизм заморозки mutable-слоёв в immutable-копии в моменты T. См.
`.claude/skills/snapshot.md`.

## Как слои связаны

```
         CLAUDE.md ──────────┐
             │               │
             ▼               ▼
         Dashboard ◄──► dept-dashboards (01, 02)
             │               │
             ├───► ADR ◄──────┐
             │      │        │
             ▼      ▼        │
         session log ◄───────┘
             │
             ▼ (копия в момент T)
         снапшот ───► memory/ (frozen)
                  └─► git-log.txt
                  └─► Dashboard.md + 01 + 02 (frozen)

         скилы ◄── (читаются Claude'ом) ── память ◄── (авто)
                                       ◄── CLAUDE.md ◄── (авто)
         агенты ◄── (spawn Claude'ом при явном вызове)
```

**Движок связности** — wiki-links Obsidian. Каждый документ ссылается на соседей и
хабы. Правила — в `.claude/skills/knowledge-graph.md`.

## Принципы

1. **Один источник правды на факт.** Статус — в Dashboard. Решение — в ADR.
   Хроника — в session log. Не дублируем.
2. **Mutable ≠ history.** Живые документы перезаписываются. История — в git и в
   снапшотах.
3. **Плотный граф.** Каждый документ имеет секцию «Связанные» с минимум 2 wiki-ссылками.
   См. `.claude/skills/knowledge-graph.md`.
4. **Аудитория > универсальность.** Dashboard общий, 01 Маркетинг — для отдела продаж
   и владельца, 02 Разработка — для разработчиков. См. `.claude/skills/multi-audience-docs.md`.
5. **Redaction.** Секретов в публичных документах нет. См. `.claude/skills/redaction.md`.
6. **ADR до кода.** Значимые решения фиксируются ДО реализации.

## Куда писать новое

| Что | Куда |
|---|---|
| Архитектурное решение | Новый ADR в `docs/40 - Architecture/42 - ADR/ADR-NNN *.md` (через `adr-drafter`) |
| Итог сессии с изменениями | Session log в `docs/70 - Meetings & Logs/` + снапшот |
| Бриф, ТЗ, требование | `docs/10 - Brief & Requirements/` |
| Аудит текущего состояния | `docs/20 - Audit/` |
| Обсуждение идеи (без решения) | `docs/90 - Ideas & Backlog/` |
| Новое процедурное правило для Claude | `.claude/skills/<name>.md` |
| Работник со scoped-правами | `.claude/agents/<name>.md` |
| Тексты, описания моделей | `docs/60 - Content/` |
| Логотип, токены, типографика | `docs/65 - Brand/` |
| Контент-план соцсетей | `docs/67 - SMM/` |
