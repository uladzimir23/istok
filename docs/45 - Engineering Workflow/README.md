# 45 — Engineering Workflow

Операционные документы по инженерному процессу. Заполняются по мере появления реальной
кодовой базы.

## Что сюда пойдёт

- `Branching model.md` — модель веток (main / dev / feat-ветки).
- `Commit conventions.md` — Conventional Commits (см. `.claude/skills/git-workflow.md`).
- `PR checklist.md` — что должно быть в PR.
- `Release flow.md` — как релизим.
- `CI.md` — что прогоняется в GitHub Actions.

## На текущей фазе

Все правила — в `.claude/skills/git-workflow.md`. Этот раздел будет заполнен после
Phase 1, когда появится стек и можно зафиксировать конкретные команды (`bun typecheck`,
`bun lint`, etc).
