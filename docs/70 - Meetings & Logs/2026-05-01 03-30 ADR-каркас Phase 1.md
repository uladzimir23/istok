## Решения принятые

- **ADR-001 PocketBase as Backend** — статус `superseded` (был proposed, пересмотрен
  до начала имплементации), супрсед ADR-005.
- **ADR-002 Next.js 15 as Frontend (proposed)** — App Router, TS strict, FSD-lite.
- **ADR-003 Brand Architecture One Site Two Brands (proposed)** — один сайт
  `istokmebel.by`, ELIS-MEBEL — раздел `/krovatki` с собственной визуальной темой
  через cascade layers.
- **ADR-004 SCSS Modules as Style Layer (proposed)** — SCSS modules + cascade layers +
  token-система (паттерн moreminsk). Tailwind/shadcn отвергнуты.
- **ADR-005 Content-as-Code Phase 1 No Backend (proposed)** — в Phase 1 backend нет;
  контент = MDX/TS + Zod-валидация. PocketBase отложен в Phase 2 при наступлении
  6 trigger-условий (второй редактор без git, обновления чаще раза в неделю, инбокс/CRM,
  sync с маркетплейсами, онлайн-заказ с конфигуратором, личный кабинет/роли).
  Supersedes ADR-001.
- **ADR-006 Self-hosted Docker VPS Hosting (proposed)** — свой VPS, docker-compose
  (web + nginx), GH Actions через ghcr.io + SSH, certbot Let's Encrypt. Паттерн
  comforthotel ADR-024.
- **ADR-007 Flat Repo Structure No Monorepo (proposed)** — плоская структура: `web/`
  (Next.js) + `content/` (MDX) + `nginx/` + `docker-compose.yml` в корне. Никаких
  workspaces / turbo.
- **Slug раздела ELIS — `/krovatki`** (отдельный ADR не нужен, деталь IA, фиксируется
  в ADR-003).
- AI-пайплайн рендеров (Gemini + Krea) — отложен из ближайшего фокуса, оформим
  отдельным ADR в Phase 2+.
