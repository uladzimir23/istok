## Что обсуждали

- Продолжение предыдущей сессии (сверка docs + GH Pages деплой). В процессе пользователь
  поставил новую задачу: **перенести istok на сервер агентства** и обновить инфу в волте
  `~/Desktop/sync-agency-server/`.
- Выяснилось, что `sync-agency-server` — это общий Hetzner-бокс `89.169.54.11` с
  каноническим паттерном деплоя (GHCR + compose + host-nginx + certbot), на котором уже
  живут 6+ сайтов агентства.
- Пользователь оспорил мой первичный выбор «эталона» (брал только moreminsk). По его
  замечанию сверил **три** проекта вместо одного — это изменило подход (см. решения).

## Решения принятые

- **ADR-009 — Hetzner Agency Server Hosting** (proposed, supersedes ADR-008). Хостинг
  istok мигрирует с GitHub Pages на сервер агентства. Триггер: сервер уже существует с
  готовым каноном — переиспользуем инфру вместо отдельного GH Pages пайплайна.
  Хостинг-цепочка: ADR-006 (VPS) → ADR-008 (GH Pages) → **ADR-009** (сервер агентства).
- **Кросс-сверка трёх проектов** (вместо слепого копирования moreminsk):
  - **moreminsk** — донор статик-паттерна (nginx:alpine раздаёт `out/`). ← модель istok.
  - **clariva** — подтвердил GHCR+ssh-обвязку (1:1), но Dockerfile standalone/Node (нужен
    `/api`) — istok'у не подходит.
  - **sync-brand-site-v2** — выпадает из канона (сборка на сервере, старый appleboy,
    секрет `SSH_PRIVATE_KEY`, без GHCR) — донором НЕ берём. Сверка спасла от копирования
    legacy.
- **istok остаётся static export** (`output: "export"`) — меняется только раздатчик:
  CDN GitHub → `nginx:alpine` за host-nginx. Заявки (`NEXT_PUBLIC_LEAD_ENDPOINT`) без изменений.
- Домен **new.istokmebel.by** (preview-поддомен, apex на Tilda до cutover), порт **3007**.
