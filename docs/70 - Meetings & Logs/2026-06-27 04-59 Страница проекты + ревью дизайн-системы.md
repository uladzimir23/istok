# Сессия Claude Code — 2026-06-27 04:59

## Что обсуждали

- Доступ к хостингу — завтра, поэтому переключились на фронт: «допилить
  оставшиеся страницы» + ревью дизайн-системы.
- Аудит показал: **единственная отсутствующая страница — `/proekty`** (битая
  ссылка из меню и футера → 404). Контентные страницы (about/dostavka/contacts/
  privacy) наполнены, не заглушки.

## Решения принятые

- **Проекты — content-as-code** (по ADR-005), а не захардкоженный массив.
- **Фиксим все 4 долга DS** (цвета→токены, инлайн→Prose, брейкпоинты→миксины).
- Брейкпоинты: кастомный набор istok (560/720/960/1100/1200px) — это «долг
  против стандарта»; мигрируем на токен-скейл sm/md/lg/xl (rem).

## Изменения в коде / docs

**Страница /proekty (feat, коммит `8b70655`):**
- `entities/project` — Zod-схема + loader (`content/projects/*.mdx`).
- `content/projects/*.mdx` — 6 проектов (мигрированы из ProjectsTeaser).
  ⚠️ Контент иллюстративный — реальные кейсы госзаказа у клиента.
- `widgets/ProjectsGrid` — серверный грид; `app/proekty` — страница.
- `ProjectsTeaser` → проп `projects` (единый источник). `validate-content`
  расширен (26 товаров + 6 проектов).

**Ревью дизайн-системы (refactor, коммит `e49f2f0`):** DS крепкая (токены
primitive→semantic, cascade layers, две бренд-темы, rem-брейкпоинты с миксинами).
Устранены долги:
- **Брейкпоинты**: 32 сырых `@media` в 15 модулях → `respond-to(...)`.
  Снаппинг к токен-скейлу. ⚠️ Пороги сместились ≤80px — нужен визуальный QA.
- **Цвета**: литералы → токены (`--color-on-accent`, `--color-bar-foreground`,
  `--on-image`); 3 намеренных media-литерала задокументированы. Бренд-синие
  остались только в lab-Hero (демо).
- **Инлайн-стили**: about/dostavka/privacy → `<Prose>`; contacts/korpusnaya →
  co-located модули. Вне `/lab` инлайнов нет.

Проверки: typecheck, build (45 страниц), validate:content — зелёные.

## Следующие шаги

1. **Визуальный QA брейкпоинтов** на границах sm/md/lg/xl (совместить с
   завтрашним ревью на сервере).
