## Стек (зафиксирован ADR-ами 2026-05-01)

| Слой | Решение | ADR |
| --- | --- | --- |
| Frontend | **Next.js 16** (App Router, TS strict, FSD-lite, static export) | ADR-002 |
| Стили | **SCSS modules** + cascade layers + token-система (паттерн moreminsk) | ADR-004 |
| Контент Phase 1 | **Content-as-code**: MDX в `content/` + Zod-валидация на билде; заявки через `NEXT_PUBLIC_LEAD_ENDPOINT` | ADR-005 |

| Структура репо | **Плоская**: `web/`, `content/`, `nginx/`, `docker-compose.yml` в корне. Без monorepo | ADR-007 |
| Бренд-архитектура | Один сайт `istokmebel.by`; ELIS = раздел `/krovatki` с cascade-layer-темой | ADR-003 |
| Аналитика | Яндекс.Метрика + GA4 + пиксели | базис |
| Формы | react-hook-form + zod | базис |
| Email | Resend | предпочтение |
| Telegram-уведомления | Telegram Bot API через server route | предпочтение |
| Backend Phase 2 | **PocketBase** (single binary, SQLite) — при наступлении trigger-условий | ADR-005 |

## Зоны кода (Phase 1 — реализовано)

```
istok/
├── web/                           # Next.js 16 (App Router, TypeScript strict, static export)
│   ├── src/
│   │   ├── app/                   # роутинг: страницы, layouts; /lab/hero и /lab/sections
│   │   ├── widgets/               # ~18 виджетов: Header, Footer, Hero, FAQ, LeadForm,
│   │   │                          #   ProductDetail, CollectionSwitcher, CategoryListing,
│   │   │                          #   ProjectsTeaser, CTABand, Certificates, ProductionProcess
│   │   ├── entities/product/      # schema.ts (Zod), loader с валидацией на билде
│   │   └── shared/design-system/  # tokens/mixins/base
│   ├── out/                       # static export (`bun run build`)
│   └── scripts/validate-content.ts
├── content/                       # контент-as-code (MDX)
│   ├── products/
│   │   ├── chairs/                # театральные кресла
│   │   ├── cabinets/              # корпусная мебель (комоды, столы, стеллажи, шкафы)
│   │   └── cribs/                 # кроватки Элис (26 опубликованных товаров всего)
│   └── projects/                  # портфолио госзаказа (пусто, .gitkeep)
├── .github/
│   └── workflows/
│       └── deploy-pages.yml       # typecheck → validate:content → build → GH Pages
└── scripts/                       # check-secrets.sh и пр.
```

(Структура зафиксирована ADR-007. `nginx/` и `docker-compose.yml` не нужны при GH Pages;
понадобятся при возврате к Docker-хостингу в Phase 2+.)

## Pre-commit / CI

**Настроено (2026-06-26):**
- `.github/workflows/deploy-pages.yml` — на push в `main`: typecheck → `bun run validate:content` → `bun run build` → deploy на GitHub Pages.
- `web/scripts/validate-content.ts` — проверяет все MDX через Zod перед сборкой.

Не настроено:
- `scripts/check-secrets.sh` — regex-сканер секретов (планировалось, см. `.claude/skills/redaction.md`).
- pre-commit hook (`lefthook`/`husky`).

## Риски и блокеры

✅ **VPS provisioning — снят.** Хостинг переведён на GitHub Pages (ADR-008).

✅ **Контент кроваток — перенесён.** 26 MDX-товаров опубликованы в `content/products/`
(chairs, cabinets, cribs).

🟡 **Приёмник заявок — заглушка.** `NEXT_PUBLIC_LEAD_ENDPOINT` не задан; LeadForm не
отправляет реальных заявок. Нужно настроить Telegram-бот + Resend и прописать endpoint
в GH Pages secrets.

🟡 **Портфолио госзаказа пусто.** `content/projects/` содержит только `.gitkeep`.
Нужен контент от клиента (кейсы, фото объектов).

🟡 **Цены «Элис» не получены.** Карточки `/krovatki/*` опубликованы без цен.
Ждём от клиента (8 моделей × 3 размера).

🟡 **DNS-cutover.** Для перехода на `istokmebel.by` нужно подключить кастомный домен
в GitHub Pages settings и подготовить DNS-чек-лист.

🟡 **AI-пайплайн не формализован.** Рендеры Gemini/Krea без пресетов и системы.
Нужен playbook (Phase 2+).
