## Зоны кода (Phase 1 — реализовано)

```
istok/
├── web/                           # Next.js 16 (App Router, TypeScript strict, static export)
│   ├── src/
│   │   ├── app/                   # роутинг: страницы, layouts; /lab/hero и /lab/sections
│   │   ├── widgets/               # ~18 виджетов: Header, Footer, Hero, FAQ, LeadForm,
│   │   │                          #   ProductDetail, CollectionSwitcher, CategoryListing,
│   │   │                          #   ProjectsTeaser, CTABand, Certificates, ProductionProcess
│   │   ├── entities/product/      # schema.ts (Zod), loader с валидацией на билде
│   │   └── shared/design-system/  # tokens/mixins/base
│   ├── out/                       # static export (`bun run build`)
│   └── scripts/validate-content.ts
├── content/                       # контент-as-code (MDX)
│   ├── products/
│   │   ├── chairs/                # театральные кресла
│   │   ├── cabinets/              # корпусная мебель (комоды, столы, стеллажи, шкафы)
│   │   └── cribs/                 # кроватки Элис (26 опубликованных товаров всего)
│   └── projects/                  # портфолио госзаказа (пусто, .gitkeep)
├── Dockerfile                     # custom multi-stage: bun deps → node build (web/+content/) → nginx:alpine
├── .dockerignore
├── infra/
│   ├── nginx/
│   │   ├── container.conf         # nginx-конфиг внутри контейнера
│   │   └── new.istokmebel.by.conf # host-vhost: proxy_pass 127.0.0.1:3007
│   └── docker-compose.yml         # сервис istok, порт 3007:80
├── .github/
│   └── workflows/
│       └── deploy.yml             # push→main → docker build → GHCR → ssh compose pull/up
└── scripts/                       # check-secrets.sh и пр.
```

(Структура зафиксирована ADR-007. `infra/` добавлен ADR-009: nginx-конфиг и compose для сервера агентства.)

- `.github/workflows/deploy.yml` — на push в `main` (пути `web/**`, `content/**`, `infra/**`, `Dockerfile`): typecheck → `bun run validate:content` → `bun run build` → docker build → GHCR → appleboy/ssh-action → compose pull/up на `89.169.54.11`. Экшены запинены к SHA.

## Риски и блокеры

✅ **VPS provisioning — снят.** Хостинг мигрирует с GitHub Pages на сервер агентства (ADR-009).

✅ **Контент кроваток — перенесён.** 26 MDX-товаров опубликованы в `content/products/`
(chairs, cabinets, cribs).

🟡 **Хостинг — ждёт серверных шагов (ADR-009).** Репо-обвязка готова (Dockerfile, `infra/`,
`deploy.yml`). Открытые блокеры: DNS A-запись `new.istokmebel.by` → `89.169.54.11`; секрет
`DEPLOY_SSH_KEY` в GitHub-репо istok; на сервере: `/opt/istok/docker-compose.yml`, host-vhost,
`certbot --nginx -d new.istokmebel.by`. Порт 3007 live не подтверждён — проверить `docker ps`.

🟡 **Приёмник заявок — заглушка.** `NEXT_PUBLIC_LEAD_ENDPOINT` не задан; LeadForm не
отправляет реальных заявок. Нужно настроить Telegram-бот + Resend и прописать endpoint
в GitHub secrets репо.

🟡 **Портфолио госзаказа пусто.** `content/projects/` содержит только `.gitkeep`.
Нужен контент от клиента (кейсы, фото объектов).

🟡 **Цены «Элис» не получены.** Карточки `/krovatki/*` опубликованы без цен.
Ждём от клиента (8 моделей × 3 размера).

🟡 **DNS-cutover.** Для перехода на `istokmebel.by` нужно переключить apex A-запись
с Tilda на `89.169.54.11` и добавить `istokmebel.by` vhost + certbot на сервере агентства.

🟡 **AI-пайплайн не формализован.** Рендеры Gemini/Krea без пресетов и системы.
Нужен playbook (Phase 2+).
